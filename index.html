Welcome to ZEV LTD

ZEV LTD is a fast-growing company in Angeles City, Balibago with more than 4 million customers globally. The organization deals with buying and selling agricultural products, operated from our headquarters. We are expanding, with a new branch planned near Balibago, and we need a network that operates independently from the headquarters.

Network Design Requirements

  1. One router and one switch to be used.
  2. Three (3) departments: Admin/IT, Finance/HR, and Customer Service/Reception.
  3. Each department needs to be in a different VLAN.
  4. Each department should have a wireless network for the users.
  5. Devices in the network should automatically receive an IPV4 address (DHCP).
  6. Devices should be able to communicate with each other.

Network Addressing

The ISP provided a base network address of 192.168.2.0/26, which will be used for the network design and implementation.

Subnetting - Part 1

Subnetting is the process of dividing a larger network into smaller, more manageable subnetworks, or subnets. It’s a technique used to improve network performance, security, and organization. Subnetting involves allocating portions of a network’s IP address space to different subnetworks, allowing for more efficient use of IP addresses and better organization of network resources.

As explained, to get the number of networks needed for this project, we look at the case study where departments are required to be created.
No. of subnets: 3 (Admin/IT, Finance/HR, and Customer service/Reception)

To find the number of subnets, this formula will be used to determine the outcome:
No. of subnets = 2n, where n represents the number of bits from the host portion.
2n = 3

Remember, you cannot use a smaller number or one equal to the required subnet, but can use a higher number to accommodate the subnet required. We cannot use 20 = 1 and 21 = 2, which are both less than 3, thus we can use 22 = 4, but we will tag it as 3. Without much explanation, let’s get the calculation done.

Basic Network: 192.168.2.0/24
Subnet Mask: 255.255.255.0 — Class C
Binary of SM: 11111111.11111111.11111111.00000000
Octet: 1st, 2nd, 3rd, 4th
Network/host: Net . Net . Net . Host

Subnetting - Part 2

To satisfy the network requirements, 2 bits from the host portion will be borrowed. Therefore, the new binary bits will be:
New Binary: 11111111.11111111.11111111.11000000
CIDR: /26
Decimal Format: 255.255.255.192

We know you will be confused about how we arrived at 192. We will show you how it’s done. This bit from 11000000 from the new subnet mask is converted to decimal format:
Solving: 1×27 + 1×26 + 0×25 + 0×24 + 0×23 + 0×22 + 0×21 + 0×20 = 128+64+0+0+0+0+0+0 = 192.

New Subnet Mask: 255.255.255.192
Block size: 64

After subnetting, we need to assign the network to each of the required networks, respectively:

Subnetting Table

Subnet Network ID Host Range Broadcast ID
First Subnet (VLAN 10) 192.168.2.0/26 192.168.2.1–192.168.2.62 192.168.2.63
Second Subnet (VLAN 20) 192.168.2.64/26 192.168.2.65–192.168.2.126 192.168.2.127
Third Subnet (VLAN 30) 192.168.2.128/26 192.168.2.129–192.168.2.190 192.168.2.191

Note: It is important to remember that the first subnet will take the base network, while the rest network ID will be determined with increments of block size, which in this case is 64, as illustrated above. The broadcast address takes the last IP of the network, while the first usable IP is the first IP after the network address, which in most cases is assigned to the default gateway.

Note: VLANs help segment network traffic, but if there are too many VLANs, it can lead to increased broadcast and multicast traffic within each VLAN. This can consume network bandwidth and processing resources on network devices, potentially slowing down network performance. To address this issue, we implemented a trunk on the gateway interface.

In networking, a trunk refers to a communication channel or link that carries multiple VLANs between switches, routers, or other network devices. Trunks are essential for enabling communication between devices in different VLANs and are commonly used in larger networks where VLAN segmentation is implemented.

Sub-Interface Creation (Inter-VLAN Routing)

We implement inter-VLAN routing to allow communication between each of the devices. This configuration will be carried on the router.

Command Descriptions
Router> enable or en To enable the router
Router# configure terminal or config Enter global configuration
Router(config)# interface g0/0 or int g0/0 Initiate the interface and port number
Router(config-if)# no shut or no shutdown Power on the interface
Router(config-if)# do wr To save the configuration
Router(config-if)# exit Exit the interface configuration mode

Inter-VLAN Routing Configuration

This process involves assigning VLANs to the default gateways for VLAN 10, VLAN 20, and VLAN 30:

Command Description
Router(config)# int g0/0.10 Assign VLAN 10 to the sub-interface
Router(config-subif)# encapsulation dot1q 10 Set encapsulation for VLAN 10
Router(config-subif)# ip add 192.168.2.1 255.255.255.192 Assign the first IP address of the first subnet
Router(config-subif)# exit Exit the sub-interface configuration
Router(config)# int g0/0.20 Assign VLAN 20 to the sub-interface
Router(config-subif)# encapsulation dot1q 20 Set encapsulation for VLAN 20
Router(config-subif)# ip add 192.168.2.65 255.255.255.192 Assign the first IP address of the second subnet
Router(config-subif)# exit Exit the sub-interface configuration
Router(config)# int g0/0.30 Assign VLAN 30 to the sub-interface
Router(config-subif)# encapsulation dot1q 30 Set encapsulation for VLAN 30
Router(config-subif)# ip add 192.168.2.129 255.255.255.192 Assign the first IP address of the third subnet
Router(config-subif)# exit Exit the sub-interface configuration
Router(config)# do wr Save the configuration

DHCP Server Configuration

As requested in the case study, the device will obtain IP automatically, and DHCP service must be enabled and configured on the router.

Admin/IT

Command Description
Router> enable or en Enables Privileged Exec Mode
Router# config t or configure terminal Enter global configuration
Router(config)# service dhcp> Enable DHCP service
Router(dhcp-config)# ip dhcp pool Admin-IT Creates a pool for Admin/IT department
Router(dhcp-config)# network 192.168.2.0 255.255.255.192 The network ID of VLAN 10
Router(dhcp-config)# default-router 192.168.2.1 Default gateway address of VLAN 10
Router(dhcp-config)# dns-server 192.168.2.1 Any IP can be assigned on a DNS server
Router(dhcp-config)# domain-name admin.com> Assigning a domain name (not compulsory)
Router(dhcp-config)# exit Exits DHCP pool configuration mode
